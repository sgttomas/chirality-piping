Define a conservative procedure for producing or reviewing the future DEL-16-04 artifacts: an agent rationale record and professional-boundary guard tests. This procedure is document-level setup guidance only; it does not implement product code or claim engineering acceptance.

## Steps

1. Confirm the work is still bounded to DEL-16-04 and SOW-070.
2. Review source authority in this order: `_REFERENCES.md` listed governing sources, `_CONTEXT.md`, approved local `Dependencies.csv`, and `execution/_Decomposition/SOFTWARE_DECOMP.md`.
3. Identify the rationale-record categories supported by sources: operation history, rationale, assumptions, affected entities, and audit metadata.
4. Mark every unsupported schema field, implementation module, or product-code path as `TBD` rather than inventing it.
5. Identify prohibited professional-boundary outputs from `docs/CONTRACT.md`, `docs/TYPES.md`, and `docs/SPEC.md`.
6. Define guard-test expectations for prohibited automatic claims and permitted software statuses.
7. Check that assumptions, warnings, limitations, and missing data remain explicit findings.
8. Check that no protected standards text, code-specific values, private project data, or private rule-pack payloads are introduced.
9. Check dependency handling against the local DAG-002 mirror; do not retire, delete, or reclassify approved ACTIVE rows during this setup pass.
10. Record unresolved issues as `TBD`, `ASSUMPTION`, or `PROPOSAL` using `docs/TYPES.md` epistemic labels.
